Phone Book Example

The main purpose of this example is to show you :

  • How to use a custom system under development
  • How Type Conversion work
  • How to initialize a list of data using the 'Setup' Interpreter
  • How to compare null value using the 'nothing' keyword
End Info
Begin Info

The PhoneBookSystemUnderDevelopment is a Fixture Factory that takes care of :

  • Importing the info.novatec.testit.livingdoc.confluence.demo.phonebook Java package. This way, there is no need to use the Import Table or the Import Macro on every specification.
  • Registering the custom converter PhoneBookEntryTypeConverter. This way, there is no need to modify our domain object with the 'Self Conversion' methods parse or toString.
  • Managing our instance of the PhoneBookFixture to make sure the instance of PhoneBook is the same for all interpreters. This is really optional but provides a way to use the same instance (singleton) of the PhoneBook that contains a list of contacts. A better approach would be to use Spring or Guice.
End Info
