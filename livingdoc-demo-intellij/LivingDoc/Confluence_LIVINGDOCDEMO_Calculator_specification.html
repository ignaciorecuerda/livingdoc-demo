Calculator Example

The main purpose of this example is to show you :

  • How to handle business rules using the 'Rule For' Interpreter
  • How to use the keyword 'error' to handle exceptions
  • How BeforeTable and AfterTable can be used to affect tables

The simplicity of the Calculator example allows to avoid confusion between the system context and the specific behavior of LivingDoc Software.
